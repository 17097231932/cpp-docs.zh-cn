# <a name="afxdll-versions"></a>AFXDLL 版本
而不是通过静态链接到 MFC 对象代码库构建你的应用程序，你可以构建你的应用程序以使用其中一个 AFXDLL 库，包含在多个正在运行的应用程序可以共享的 DLL 的 MFC。 AFXDLL 名称的表，请参阅[Dll： 命名约定](../build/naming-conventions-for-mfc-dlls.md)。  
  
> [!NOTE]
>  默认情况下，MFC 应用程序向导创建 AFXDLL 项目。 若要使用静态链接的 MFC 代码相反，设置**静态库中使用 MFC** MFC 应用程序向导中的选项。 静态链接不在标准版本的 Visual c + + 中可用。
