# <a name="c-runtime-error-r6002"></a>C 运行时错误 R6002

未加载的浮点支持

未链接所必需的浮点库。

> [!NOTE]
> 如果运行应用时遇到此错误消息，该应用已关闭，因为它具有内部问题。 有多种原因引起此错误，但通常导致应用程序的代码中有缺陷，或尝试运行的应用程序不为特定计算机处理器生成。
>
> 可以尝试以下步骤来修复此错误：
>
> - 使用**应用程序和功能**或**程序和功能**页**控制面板**来修复或重新安装该程序。
> - 检查**Windows Update**中**控制面板**的软件更新。
> - 检查应用程序的更新版本。 如果问题仍然存在，请与应用供应商联系。

**程序员提供的的信息**

不链接的浮点库时，可以在应用中发生此错误。 检查这些原因之一：

- 格式字符串`printf_s`或`scanf_s`函数包含浮点格式规范中，并且该程序未包含任何浮点值或变量。 若要解决此问题，使用浮点型参数对应于浮点格式规范中，或在程序中的其他地方执行浮点赋值。 这会导致要加载的浮点支持。

- 编译器通过加载浮点支持仅在必要时，最小化程序的大小。 编译器无法检测到浮点运算或浮点格式规范中格式字符串，因此它不会加载需要浮点例程。 若要解决此问题，使用浮点格式规范和提供浮点型参数，或在程序中的其他地方执行浮点赋值。 这会导致要加载的浮点支持。

- 在混合语言程序中，当该程序进行链接时在 FORTRAN 库之前指定的 C 库。 重新链接和上一次指定的 C 库。
