# CA0058
无法加载所引用的程序集。  
  
 如果在分析时，分析工具无法加载程序集所引用的程序集，则会出现此错误。  因此，该错误可能会在其他位置导致 CA0001，因为在发生此错误时可能会出现许多意外的状态。  如果从 [!INCLUDE[vsprvs](../assembler/masm/includes/vsprvs_md.md)] 中使用托管代码分析工具，则下面列出的原因可能会导致 CA0058：  
  
-   输入程序集已被重写，现在所指向的程序集引用 [!INCLUDE[vsprvs](../assembler/masm/includes/vsprvs_md.md)] 项目文件中引用列表外部的其他程序集。  
  
-   如果对项目进行修改以便在 FxCopCmd.exe 运行之前调用自定义生成步骤，而且自定义生成过程会添加更多的输入程序集引用，则会生成 CA0058 错误。  
  
-   在 C\+\+ 中，可以这样来引用程序集：使用 VCPROJ 中的 ForcedUsing，并重写命令行以传递 \/ ForcedUsing，或者仅在源代码中添加 \#using \<Some.dll\>。  
  
 在所有情况下，对此问题的解决方案都是将缺少的引用添加到 [!INCLUDE[vsprvs](../assembler/masm/includes/vsprvs_md.md)] 项目本身。
