# <a name="remote-automation"></a>远程自动化
> [!NOTE]
>  远程自动化是过时的技术，不应该用于新应用程序。 Visual c + + 不支持 Windows 95。 在其中你将需要支持远程自动化的情况下中所述[其中不远程自动化适用于](where-does-remote-automation-fit-in-q.md)。  
  
 远程自动化是一种[自动化](../mfc/automation.md)这样接口使用者能够在网络上执行的接口提供程序，例如，驻留在另一台计算机上。  
  
 本文说明了如何创建可远程调用和执行的自动化对象，以及如何创建可使用这些远程自动化对象的自动化控制器。 它还检查配置选项并指出远程自动化和 DCOM（COM 和 OLE 的分布式版本，允许远程调用和执行除了与自动化相关的接口之外的接口）之间的主要差异。
