# <a name="cmfcvisualmanager-class"></a>CMFCVisualManager 类
提供在全局级别更改应用程序外观的支持。 `CMFCVisualManager` 类与说明如何绘制应用程序中 GUI 控件的类一起使用，且使用一致样式。 这些其他类称为视觉管理器，从 `CMFCBaseVisualManager`继承而来。  
  
## <a name="syntax"></a>语法  
  
```  
class CMFCVisualManager : public CMFCBaseVisualManager  
```

## <a name="remarks"></a>备注  
 因为`CMFCVisualManager`类可控制应用程序的 GUI，每个应用程序可以具有的任一一个实例`CMFCVisualManager`，或从派生类的一个实例`CMFCVisualManager`。 您的应用程序还可以有效，无需`CMFCVisualManager`。 使用静态方法`GetInstance`以获取指向当前`CMFCVisualManager`-派生的对象。  
  
 若要更改您的应用程序的外观必须使用其他类提供用于绘制您的应用程序的可视元素的所有方法。 这些类的一些示例包括[CMFCVisualManagerOfficeXP 类](../../mfc/reference/cmfcvisualmanagerofficexp-class.md)， [CMFCVisualManagerOffice2003 类](../../mfc/reference/cmfcvisualmanageroffice2003-class.md)，和[CMFCVisualManagerOffice2007 类](../../mfc/reference/cmfcvisualmanageroffice2007-class.md)。 当您想要更改您的应用程序的外观时，将这些视觉管理器之一传递给方法`SetDefaultManager`。 有关演示如何应用程序可模拟 Microsoft Office 2003 外观的示例，请参阅[CMFCVisualManagerOffice2003 类](../../mfc/reference/cmfcvisualmanageroffice2003-class.md)。  
  
 所有绘制方法都是虚拟的。 这使您能够为您的应用程序的图形用户界面创建自定义的视觉样式。 如果您想要创建您自己的可视样式，从视觉管理器类之一派生一个类并重写绘制您想要更改的方法。
