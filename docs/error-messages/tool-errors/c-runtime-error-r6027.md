# <a name="c-runtime-error-r6027"></a>C 运行时错误 R6027

空间不足，无法进行 lowio 初始化

> [!NOTE]
> 如果在运行应用程序时遇到此错误消息，则该应用程序已关闭，因为它具有内部内存问题。 此错误有几个可能的原因，但通常是由于内存不足造成的。 它也可能是由应用程序中的 bug、其使用的可视化C++库损坏或驱动程序引起的。
>
> 可以尝试以下步骤来修复此错误：
>
> - 关闭其他正在运行的应用程序或重新启动计算机以释放内存。
> - 使用**控制面板**中的 "**应用和功能**" 或 "**程序和功能**" 页来修复或重新安装该程序。
> - 如果应用在最近一次安装其他应用或驱动程序之前工作，请使用**控制面板**中的 "**应用和功能**" 或 "**程序和功能**" 页删除新应用或驱动程序，然后重试应用。
> - 使用**控制面板**中的C++ "**应用和功能**" 或 "**程序和功能**" 页来修复或重新安装 Microsoft Visual 可再发行组件的所有副本。
> - 检查 "**控制面板**" 中的 "软件更新" **Windows 更新**。
> - 检查应用的更新版本。 如果问题仍然存在，请与应用程序供应商联系。

**面向程序员的信息**

如果没有足够的可用内存用于初始化 C 运行时中的低级别 i/o 支持，则会出现此错误。 此错误通常在应用程序启动时出现。 验证应用以及它所加载的驱动程序和 dll 在启动时不会损坏堆。
