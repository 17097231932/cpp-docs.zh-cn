# <a name="cdockablepane-class"></a>CDockablePane Class

```

```

## <a name="remarks"></a>备注  
 `CDockablePane` 实现以下功能：  
  
-   到主框架窗口停靠窗格。  
  
-   切换到自动隐藏模式的窗格。  
  
-   附加到选项卡式窗口的窗格。  
  
-   浮点袖珍框架窗口的窗格中。  
  
-   到浮点袖珍框架窗口中的另一个窗格停靠窗格。  
  
-   调整大小的窗格。  
  
-   加载和保存停靠的窗格中的状态。  
  
    > [!NOTE]
    >  状态信息保存到 Windows 注册表中。  
  
-   创建一个窗格与或无标题。 标题可以文本标签，并且它可以填充渐变颜色。  
  
-   显示窗格的内容的同时拖动一个窗格  
  
-   显示拖动矩形的同时拖动窗格。  
  
 若要在你的应用程序中使用停靠的窗格中，派生从你窗格类`CDockablePane`类。 到主框架窗口对象或控制您的窗格的实例的窗口对象嵌入派生的对象。 然后调用[CDockablePane::Create](#create)方法或[cdockablepane:: Createex](#createex)方法处理主框架窗口中的 WM_CREATE 消息时。 最后，通过调用设置窗格对象[CBasePane::EnableDocking](../../mfc/reference/cbasepane-class.md#enabledocking)， [cbasepane:: Dockpane](../../mfc/reference/cbasepane-class.md#dockpane)，或[cdockablepane:: Attachtotabwnd](#attachtotabwnd)。  
  
## <a name="customization-tips"></a>自定义提示  
 以下提示适用于`CDockablePane`对象：  
  
-   如果调用[cdockablepane:: Attachtotabwnd](#attachtotabwnd)对于两个非选项卡式、 可停靠窗格，指向选项卡式窗口的指针将返回在`ppTabbedControlBar`参数。 你可以继续使用此参数将选项卡添加到选项卡式窗口。  
  
-   通过创建的选项卡式窗格种[cdockablepane:: Attachtotabwnd](#attachtotabwnd)由`CDockablePane`对象在*pTabControlBarAttachTo*参数。 你可以调用[cdockablepane:: Settabbedpanertc](#settabbedpanertc)到设置的选项卡式窗格类型`CDockablePane`将创建。 默认类型由`dwTabbedStyle`的[CDockablePane::Create](#create)当你首次创建`CDockablePane`。 如果*dwTabbedStyle*是默认的类型是的 AFX_CBRS_OUTLOOK_TABS [CMFCOutlookBar 类](../../mfc/reference/cmfcoutlookbar-class.md); 如果*dwTabbedStyle* AFX_CBRS_REGULAR_TABS 默认类型为[CTabbedPane 类](../../mfc/reference/ctabbedpane-class.md)。  
  
-   如果你想要将一个可停靠窗格停靠到另一个，调用[CDockablePane::DockToWindow](#docktowindow)方法。 原始窗格中必须停靠位置然后才能调用此方法。  
  
-   成员变量[CDockablePane::m_bHideInAutoHideMode](#m_bhideinautohidemode)控件如何可停靠的窗格的行为方式中自动隐藏模式下在调用时[CDockablePane::ShowPane](#showpane)。 如果此成员变量设置为`TRUE`，可停靠的窗格和其自动隐藏按钮将处于隐藏状态。 否则，它们将滑入和签出。  
  
-   你可以通过设置来禁用自动隐藏动画[CDockablePane::m_bDisableAnimation](#m_bdisableanimation)到成员变量`TRUE`。

##  <a name="attachtotabwnd"></a>  Cdockablepane:: Attachtotabwnd  
 将当前窗格附加到目标窗格中，创建选项卡式的窗格。  
  
```  
virtual CDockablePane* AttachToTabWnd(
    CDockablePane* pTabControlBarAttachTo,  
    AFX_DOCK_METHOD dockMethod,  
    BOOL bSetActive= TRUE,  
    CDockablePane** ppTabbedControlBar = NULL);  
```  
  
### <a name="parameters"></a>参数  
 [in][out]*pTabControlBarAttachTo*  
 指定当前窗格中将附加到目标窗格。 目标窗格中必须是可停靠的窗格。  
  
 [in]*dockMethod*  
 指定的停靠的方法。  
  
 [in]*bSetActive*  
 `TRUE` 若要在附加操作; 后激活选项卡式的窗格否则为`FALSE`。  
  
 [out]*ppTabbedControlBar*  
 包含附加操作生成的选项卡式的窗格。  
  
### <a name="return-value"></a>返回值  
 指向当前窗格中，如果它不是选项卡式的窗格中;否则为指向附加操作生成的选项卡式窗格的指针。 返回值是`NULL`或如果无法附加当前窗格中，如果发生错误。  
  
### <a name="remarks"></a>备注  
 当一个可停靠的窗格中将附加到使用此方法的另一个窗格中时，发生以下情况：  
  
1.  Framework 检查是否目标窗格中*pTabControlBarAttachTo*正则表达式停靠窗格或如果它派生自[CBaseTabbedPane](../../mfc/reference/cbasetabbedpane-class.md)。  
  
2.  如果目标窗格中，选项卡式的窗格框架将当前窗格中向其添加为一个选项卡。  
  
3.  如果目标窗格中，正则停靠窗格框架创建选项卡式的窗格。  
  
    -   框架调用`pTabControlBarAttachTo->CreateTabbedPane`。 取决于新的选项卡式窗格中的样式`m_pTabbedControlBarRTC`成员。 默认情况下，此成员设置为运行时类的[CTabbedPane](../../mfc/reference/ctabbedpane-class.md)。 如果你通过`AFX_CBRS_OUTLOOK_TABS`样式作为*dwTabbedStyle*参数[CDockablePane::Create](#create)方法，运行时类对象设置为运行时类的[CMFCOutlookBar](../../mfc/reference/cmfcoutlookbar-class.md). 你可以随时更改新窗格中的样式更改此成员。  
  
    -   当此方法创建选项卡式的窗格时，框架将替换指向的指针`pTabControlBarAttachTo`（窗格中是否停靠或浮动在多袖珍框架窗口中） 用一个指针指向新的选项卡式窗格。  
  
    -   框架就会将`pTabControlBarAttachTo`到选项卡式窗格中，作为第一个选项卡的窗格。然后，框架将当前窗格添加为第二个选项卡。  
  
4.  如果当前窗格派生自`CBaseTabbedPane`，所有其选项卡移动到`pTabControlBarAttachTo`当前窗格中将被破坏。 因此，要小心时调用此方法，因为该方法返回指向当前窗格的指针可能无效。  
  
 如果生成停靠布局时，可附加到另一个窗格中，设置`dockMethod`到`DM_SHOW`。  
  
 另一个窗格附加到它之前，应将停靠的第一个窗格。

##  <a name="getdockingstatus"></a>  CDockablePane::GetDockingStatus  
 确定一个窗格停靠的能力根据提供的指针的位置。  
  
```  
virtual AFX_CS_STATUS GetDockingStatus(
    CPoint pt,  
    int nSensitivity);
```  
  
### <a name="parameters"></a>参数  
 [in]*pt*  
 以屏幕坐标表示指针的位置。  
  
 [in]*nSensitivity*  
 距离，以像素为单位，离开矩形的边缘指针必须是以启用停靠。  
  
### <a name="return-value"></a>返回值  
 以下状态值之一：  
  
|`AFX_CS_STATUS` 值|含义|  
|---------------------------|-------------|  
|`CS_NOTHING`|指针不是停靠站点段。 框架将窗格未停靠。|  
|`CS_DOCK_IMMEDIATELY`|指针位于转移停靠站点中即时模式 (窗格中使用`DT_IMMEDIATE`停靠模式)。 框架立即停靠窗格。|  
|`CS_DELAY_DOCK`|指针位于停靠站点是另一个停靠窗格或主框架边缘。 框架将在延迟后停靠窗格。 请参阅有关这种延迟的详细信息备注部分。|  
|`CS_DELAY_DOCK_TO_TAB`|指针位于转移使窗格停靠在选项卡式窗口停靠站点。 当指针位于另一个停靠窗格的标题或选项卡式窗格的选项卡区域上方时，将发生这种情况。|  
  
### <a name="remarks"></a>备注  
 框架调用此方法以处理停靠浮动窗格。  
  
 浮动工具栏或停靠的窗格，使用`DT_IMMEDIATE`停靠模式下，框架会延迟停靠命令，以使用户能够移出的父框架的工作区的窗口，停靠发生之前。 延迟的长度以毫秒为单位进行并由控制[CDockingManager::m_nTimeOutBeforeToolBarDock](../../mfc/reference/cdockingmanager-class.md#m_ntimeoutbeforetoolbardock)数据成员... 默认值[CDockingManager::m_nTimeOutBeforeToolBarDock](../../mfc/reference/cdockingmanager-class.md#m_ntimeoutbeforetoolbardock)为 200。 此行为模拟的停靠行为[!INCLUDE[ofprword](../../mfc/reference/includes/ofprword_md.md)]2007年。  
  
 有关延迟停靠状态 (`CS_DELAY_DOCK`和`CS_DELAY_DOCK_TO_TAB`)，框架不会执行停靠直到用户释放鼠标按钮。 如果窗格使用`DT_STANDARD`停靠模式下，框架在预计的停靠位置显示一个矩形。 如果窗格使用`DT_SMART`停靠模式下，框架显示智能停靠标记和半透明的矩形的预计的停靠位置。 若要指定您的窗格的停靠模式，请调用[CBasePane::SetDockingMode](../../mfc/reference/cbasepane-class.md#setdockingmode)方法。 有关智能停靠的详细信息，请参阅[CDockingManager::GetSmartDockingParams](../../mfc/reference/cdockingmanager-class.md#getsmartdockingparams)。

##  <a name="undockpane"></a>  CDockablePane::UndockPane  
 中取消停靠窗格中，从主框架窗口或袖珍框架窗口容器。  
  
```

```  
  
### <a name="parameters"></a>参数  
 [in]*bDelay*  
 `TRUE` 延迟计算停靠布局;`FALSE`立即重新计算停靠布局。
