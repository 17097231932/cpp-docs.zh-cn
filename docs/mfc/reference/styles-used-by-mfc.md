# <a name="styles-used-by-mfc"></a>MFC 使用的样式

在创建相应的 MFC 对象时，请使用以下样式标志指定窗口或控件外观和行为。 在大多数情况下，这些样式设置在类`Create`函数的*dwStyle*参数中。

## <a name="button-styles"></a><a name="button-styles"></a>按钮样式

按钮样式适用于[CButton 类](../../mfc/reference/cbutton-class.md)对象，如单选按钮、复选框和按钮。 在[CButton：：：创建](../../mfc/reference/cbutton-class.md#create)的*dwStyle*参数中指定样式的组合。 有关 Windows 中的按钮样式的详细信息，请参阅[按钮样式 （Windows）](/windows/win32/Controls/button-styles)。

### <a name="button-types"></a>按钮类型

下表列出了按钮类型。 您可以选择以下选项之一。 如果不指定按钮类型，则默认值为BS_PUSHBUTTON。

|类型|说明|
|----------|-----------------|
|BS_3STATE|创建一个复选框按钮，该按钮具有三种状态：BST_CHECKED、BST_INDETERMINATE和BST_UNCHECKED。 单击该按钮会向所有者窗口发送BN_CLICKED通知，但不会更改按钮的状态。 默认情况下，关联的文本显示在复选框的右侧。 要将文本显示在复选框的左侧，请使用BS_LEFTTEXT或BS_RIGHTBUTTON样式。|
|BS_AUTO3STATE|创建一个复选框按钮，该按钮具有三种状态：BST_CHECKED、BST_INDETERMINATE和BST_UNCHECKED。 单击该按钮会向所有者窗口发送BN_CLICKED通知并更改按钮的状态。 该按钮按BST_CHECKED、BST_INDETERMINATE和BST_UNCHECKED的顺序表示循环。 默认情况下，关联的文本显示在复选框的右侧。 要将文本显示在复选框的左侧，请使用BS_LEFTTEXT或BS_RIGHTBUTTON样式。|
|BS_AUTOCHECKBOX|创建一个复选框按钮，该按钮具有两种状态：BST_CHECKED和BST_UNCHECKED。 单击该按钮会向所有者窗口发送BN_CLICKED通知并更改按钮的状态。 默认情况下，关联的文本显示在复选框的右侧。 要将文本显示在复选框的左侧，请使用BS_LEFTTEXT或BS_RIGHTBUTTON样式。|
|BS_AUTORADIOBUTTON|创建具有两种状态的单选按钮：BST_CHECKED和BST_UNCHECKED。 单选按钮通常以组形式使用，每个组一次最多有一个选中的选项。 单击该按钮会向所有者窗口发送BN_CLICKED通知，将单击的单选按钮的状态设置为BST_CHECKED，并将按钮组中所有其他单选按钮的状态设置为BST_UNCHECKED。 默认情况下，关联的文本显示在单选按钮的右侧。 要在单选按钮的左侧显示文本，请使用BS_LEFTTEXT或BS_RIGHTBUTTON样式。|
|BS_CHECKBOX|创建一个复选框按钮，该按钮具有两种状态：BST_CHECKED和BST_UNCHECKED。 单击该按钮会向所有者窗口发送BN_CLICKED通知，但不会更改按钮的状态。 默认情况下，关联的文本显示在复选框的右侧。 要将文本显示在复选框的左侧，请使用BS_LEFTTEXT或BS_RIGHTBUTTON样式。|
|BS_COMMANDLINK|创建命令链接按钮。 命令链接按钮是特定于 Windows Vista 的命令按钮，在主文本的左侧显示一个绿色箭头，并在主文本下方显示注释。 您可以使用[CButton：：setNote](../../mfc/reference/cbutton-class.md#setnote)设置注释文本。|
|BS_DEFCOMMANDLINK|创建命令链接按钮。 命令链接按钮是特定于 Windows Vista 的命令按钮，在主文本的左侧显示一个绿色箭头，并在主文本下方显示注释。 您可以使用[CButton：：setNote](../../mfc/reference/cbutton-class.md#setnote)设置注释文本。 如果按钮在对话框中，则按下 ENTER 键会向对话框发送BN_CLICKED通知，即使该按钮没有输入焦点也是如此。|
|BS_DEFPUSHBUTTON|创建具有粗黑色边框的命令按钮。 如果按钮在对话框中，则按下 ENTER 键会向对话框发送BN_CLICKED通知，即使该按钮没有输入焦点也是如此。|
|BS_DEFSPLITBUTTON|创建拆分按钮。 拆分按钮是特定于 Windows Vista 的命令按钮，其中包含与下拉箭头相邻的按钮。 单击该按钮时，将执行默认命令。 单击下拉箭头时，将显示其他命令的菜单。 如果拆分按钮位于对话框中，则按下 ENTER 键也会向对话框发送BN_CLICKED通知，即使该按钮没有输入焦点|
|BS_GROUPBOX|创建一个矩形，其中可以对其他按钮进行分组。 与此样式关联的文本显示在矩形的左上角。|
|BS_OWNERDRAW|创建所有者绘制的按钮。 当按钮的可视`DrawItem`方面已更改时，框架将调用方法。 使用`CBitmapButton`类时必须设置此样式。|
|BS_PUSHBUTTON|创建一个命令按钮，在用户单击该按钮时向所有者窗口发送BN_CLICKED通知。|
|BS_RADIOBUTTON|创建具有两种状态的单选按钮：BST_CHECKED和BST_UNCHECKED。 单选按钮通常以组形式使用，每个组一次最多有一个选中的选项。 单击该按钮会向所有者窗口发送BN_CLICKED通知，但不会自动更改组中任何按钮的状态。 默认情况下，关联的文本显示在单选按钮的右侧。 要在单选按钮的左侧显示文本，请使用BS_LEFTTEXT或BS_RIGHTBUTTON样式。|
|BS_SPLITBUTTON|创建拆分按钮。 拆分按钮是特定于 Windows Vista 的命令按钮，其中包含与下拉箭头相邻的按钮。 单击该按钮时，将执行默认命令。 单击下拉箭头时，将显示其他命令的菜单。|
|BS_USERBUTTON|已过时，但提供了与 16 位版本的 Windows 的兼容性。 基于 Win32 的应用程序应该使用BS_OWNERDRAW。|

### <a name="radio-button-and-check-box-styles"></a>单选按钮和复选框样式

下表列出了特定于单选按钮和复选框的样式。 在所有其他按钮类型中，这些样式将被忽略。 您可以选择以下一种或多项。

|样式|说明|
|-----------|-----------------|
|BS_LEFTTEXT|与单选按钮或复选框样式结合使用时，文本将显示在单选按钮或复选框的左侧。|
|BS_RIGHTBUTTON|与单选按钮或复选框样式结合使用时，文本将显示在单选按钮或复选框的左侧。 此样式与BS_LEFTTEXT样式相同。|
|BS_PUSHLIKE|使复选框或单选按钮的外观和操作类似于命令按钮。 当该按钮的状态BST_CHECKED时，该按钮在BST_INDETERMINATE状态时按下并变暗，并在其状态BST_UNCHECKED时释放。|

### <a name="button-text-alignment-styles"></a>按钮文本对齐样式

下表列出了水平和垂直文本对齐选项。 您可以选择以下选项之一。

|样式|说明|
|-----------|-----------------|
|BS_LEFT|左侧对齐按钮矩形中的文本。 但是，如果按钮是一个复选框或单选按钮，没有BS_RIGHTBUTTON样式，则文本在复选框或单选按钮的右侧保持对齐。|
|BS_RIGHT|右对齐按钮矩形中的文本。 但是，如果按钮是一个复选框或单选按钮，没有BS_RIGHTBUTTON样式，则文本在复选框或单选按钮的右侧右对齐。|
|BS_CENTER|在按钮矩形中水平居中文本。|
|BS_TOP|将文本放在按钮矩形的顶部。|
|BS_BOTTOM|将文本放在按钮矩形的底部。|
|BS_VCENTER|在按钮矩形中垂直居中文本。|

### <a name="button-content-options"></a>按钮内容选项

下表列出了指示按钮中显示的选项。 仅显示文本的按钮类型将忽略这些样式。 您可以选择以下选项之一。

|样式|说明|
|-----------|-----------------|
|BS_BITMAP|指定按钮显示位图。|
|BS_ICON|指定按钮显示图标。|
|BS_TEXT|指定按钮显示文本。|

### <a name="other-button-options"></a>其他按钮选项

下表列出了可用于任何按钮类型的其他选项。 您可以选择以下一种或多项。

|样式|说明|
|-----------|-----------------|
|BS_FLAT|指定按钮是二维的，并且不会使用默认的图带绘制以创建三维图像。|
|BS_MULTILINE|如果文本字符串太长，无法位于按钮矩形中的一行，则将按钮文本换行到多行。|
|BS_NOTIFY|启用按钮，将BN_DBLCLK、BN_KILLFOCUS和BN_SETFOCUS通知消息发送到其父窗口。 请注意，无论是否指定此样式，按钮都会发送BN_CLICKED通知。|

## <a name="combo-box-styles"></a><a name="combo-box-styles"></a>组合盒样式

以下组合框样式在 MFC 中可用。 有关 Windows 中的组合框样式的详细信息，请参阅[组合框样式 （Windows）。](/windows/win32/Controls/combo-box-styles)

|样式|说明|
|-----------|-----------------|
|CBS_AUTOHSCROLL|当用户在行尾键入字符时，自动将编辑控件中的文本向右滚动。 如果未设置此样式，则仅允许矩形边界内的文本。|
|CBS_DISABLENOSCROLL|当列表框不包含足够的项目以滚动时，列表框将显示禁用的垂直滚动条。 如果不使用此样式，则当列表框未包含足够的项时，将会隐藏滚动条。|
|CBS_DROPDOWN|与CBS_SIMPLE类似，除非用户选择编辑控件旁边的图标，否则不会显示列表框。|
|CBS_DROPDOWNLIST|与CBS_DROPDOWN类似，只不过编辑控件替换为在列表框中显示当前选择的静态文本项。|
|CBS_HASSTRINGS|所有者绘制组合框包含由字符串组成的项。 组合框维护字符串的内存和指针，以便应用程序可使用 `GetText` 成员函数检索特定项的文本。|
|CBS_LOWERCASE|转换为小写选择字段和列表中的所有文本。|
|CBS_NOINTEGRALHEIGHT|指定组合框的大小与应用程序创建组合框时指定的大小完全相同。 通常，Windows 会调整组合框的大小，以使组合框不会仅显示部分项。|
|CBS_OEMCONVERT|在组合框编辑控件中输入的文本将从 ANSI 字符集转换为 OEM 字符集，然后转换回 ANSI。 这可确保当应用程序调用 `AnsiToOem` Windows 函数将组合框中的 ANSI 字符串转换为 OEM 字符时，能正确进行字符转换。 此样式最适用于包含文件名的组合框，且仅适用于使用 CBS_SIMPLE 或 CBS_DROPDOWN 样式创建的组合框。|
|CBS_OWNERDRAWFIXED|列表框的所有者负责绘制其内容;列表框中的项目都具有相同的高度。|
|CBS_OWNERDRAWVARIABLE|列表框的所有者负责绘制其内容;列表框中的项目高度是可变的。|
|CBS_SIMPLE|列表框随时显示。 列表框中的当前选择显示在编辑控件中。|
|CBS_SORT|自动对输入到列表框中的字符串进行排序。|
|CBS_UPPERCASE|转换为大写，选择字段和列表中的所有文本。|

## <a name="edit-styles"></a><a name="edit-styles"></a>编辑样式

编辑样式适用于[CEdit 类](../../mfc/reference/cedit-class.md)对象。 在[CEdit：：create](../../mfc/reference/cedit-class.md#create)的*dwStyle*参数中指定样式的组合。 有关 Windows 中编辑控件样式的详细信息，请参阅[编辑控件样式 （Windows）。](/windows/win32/Controls/edit-control-styles)

|样式|说明|
|-----------|-----------------|
|ES_AUTOHSCROLL|当用户在行尾键入字符时，自动将文本向右滚动 10 个字符。 当用户按下 ENTER 键时，控件将所有文本滚动回位置 0。|
|ES_AUTOVSCROLL|当用户在最后一行按 ENTER 时，自动向上滚动一页文本。|
|ES_CENTER|将文本居中，以单行或多行编辑控件为中心。|
|ES_LEFT|在单行或多行编辑控件中左对齐文本。|
|ES_LOWERCASE|将所有字符转换为小写，因为它们被键入到编辑控件中。|

|WS_CLIPSIBLINGS|剪辑子窗口彼此;也就是说，当特定子窗口收到绘制消息时，WS_CLIPSIBLINGS样式将所有其他重叠的子窗口从要更新的子窗口区域中夹出。 （如果未给出WS_CLIPSIBLINGS，并且子窗口重叠，则当您在子窗口的工作区中绘制时，可以在相邻子窗口的工作区中绘制。仅适用于WS_CHILD样式。|
|WS_DISABLED|创建初始禁用的窗口。|
|WS_DLGFRAME|创建具有双边框但没有标题的窗口。|
|WS_GROUP|指定一组控件的第一个控件，用户可以在其中使用箭头键从一个控件移动到下一个控件。 在第一个控件属于同一组后，使用 WS_GROUP 样式 FALSE 定义的所有控件。 具有WS_GROUP样式的下一个控件将启动下一个组（即，一个组结束下一个组开始的位置）。|
|WS_HSCROLL|创建具有水平滚动条的窗口。|
|WS_ICONIC|创建最初最小化的窗口。 与WS_MINIMIZE风格相同。|
|WS_MAXIMIZE|创建最大大小的窗口。|
|WS_MAXIMIZEBOX|创建具有"最大化"按钮的窗口。|
|WS_MINIMIZE|创建最初最小化的窗口。 仅适用于WS_OVERLAPPED样式。|
|WS_MINIMIZEBOX|创建具有"最小化"按钮的窗口。|
|WS_OVERLAPPED|创建重叠窗口。 重叠的窗口通常具有标题和边框。|
|WS_OVERLAPPEDWINDOW|创建具有WS_OVERLAPPED、WS_CAPTION、WS_SYSMENU、WS_THICKFRAME、WS_MINIMIZEBOX和WS_MAXIMIZEBOX样式的重叠窗口。|
|WS_POPUP|创建弹出窗口。 不能与WS_CHILD样式一起使用。|
|WS_POPUPWINDOW|创建具有WS_BORDER、WS_POPUP和WS_SYSMENU样式的弹出窗口。 WS_CAPTION样式必须与WS_POPUPWINDOW样式相结合，以使"控件"菜单可见。|
|WS_SIZEBOX|创建具有大小调整边框的窗口。 与WS_THICKFRAME风格相同。|
|WS_SYSMENU|创建标题栏中包含"控制"菜单框的窗口。 仅用于带标题栏的窗口。|
|WS_TABSTOP|指定用户可以使用 TAB 键移动的任意数量的控件之一。 TAB 键将用户移动到WS_TABSTOP样式指定的下一个控件。|
|WS_THICKFRAME|创建具有粗框架的窗口，可用于调整窗口的大小。|
|WS_TILED|创建重叠窗口。 重叠的窗口具有标题栏和边框。 与WS_OVERLAPPED风格相同。|
|WS_TILEDWINDOW|创建具有WS_OVERLAPPED、WS_CAPTION、WS_SYSMENU、WS_THICKFRAME、WS_MINIMIZEBOX和WS_MAXIMIZEBOX样式的重叠窗口。 与WS_OVERLAPPEDWINDOW风格相同。|
|WS_VISIBLE|创建初始可见的窗口。|
|WS_VSCROLL|创建具有垂直滚动条的窗口。|

## <a name="extended-window-styles"></a><a name="extended-window-styles"></a>扩展窗口样式

扩展窗口样式适用于[CWnd 类](../../mfc/reference/cwnd-class.md)对象。 在 CWnd 的*dwExStyle*参数中指定样式的组合[：：createEx](../../mfc/reference/cwnd-class.md#createex)。 有关 Windows 中扩展窗口样式的详细信息，请参阅[扩展窗口样式 （Windows）。](/windows/win32/winmsg/extended-window-styles)

|样式|说明|
|-----------|-----------------|
|WS_EX_ACCEPTFILES|指定使用此样式创建的窗口接受拖放文件。|
|WS_EX_APPWINDOW|当窗口可见时，将顶层窗口强制强制到任务栏上。|
|WS_EX_CLIENTEDGE|指定窗口具有 3D 外观 ，即具有凹陷边缘的边框。|
|WS_EX_CONTEXTHELP|在窗口的标题栏中包含一个问号。 当用户单击问号时，光标将变为带有指针的问号。 这时，如果用户单击子窗口，子窗口将收到 WM_HELP 消息。|
|WS_EX_CONTROLPARENT|允许用户使用 TAB 键在窗口的子窗口之间导航。|
|WS_EX_DLGMODALFRAME|指定具有双边框的窗口，当您在*dwStyle*参数中指定WS_CAPTION样式标志时，可以使用标题栏创建该窗口（可选）。|
|WS_EX_LAYERED|窗口是[分层窗口](/windows/win32/winmsg/window-features)。 如果窗口的[类样式](/windows/win32/winmsg/about-window-classes)为CS_OWNDC或CS_CLASSDC，则无法使用此样式。 但是，Microsoft Windows 8 确实支持子窗口的WS_EX_LAYERED样式，其中以前的 Windows 版本仅支持顶级窗口。|
|WS_EX_LEFT|提供窗口泛型左对齐属性。 这是默认值。|
|WS_EX_LEFTSCROLLBAR|将垂直滚动条放在工作区的左侧。|
|WS_EX_LTRREADING|使用从左到右的阅读顺序属性显示窗口文本。 这是默认值。|
|WS_EX_MDICHILD|创建 MDI 子窗口。|
|WS_EX_NOPARENTNOTIFY|指定使用此样式创建的子窗口在创建或销毁子窗口时不会将WM_PARENTNOTIFY消息发送到其父窗口。|
|WS_EX_OVERLAPPEDWINDOW|结合WS_EX_CLIENTEDGE和WS_EX_WINDOWEDGE风格|
|WS_EX_PALETTEWINDOW|结合了WS_EX_WINDOWEDGE和WS_EX_TOPMOST风格。|
|WS_EX_RIGHT|提供窗口泛型右对齐属性。 这依赖于窗口类。|
|WS_EX_RIGHTSCROLLBAR|将垂直滚动条（如果存在）放在工作区的右侧。 这是默认值。|
|WS_EX_RTLREADING|使用从右到左读取顺序属性显示窗口文本。|
|WS_EX_STATICEDGE|创建具有三维边框样式的窗口，用于不接受用户输入的项。|
|WS_EX_TOOLWINDOW|创建工具窗口，该窗口是用作浮动工具栏的窗口。 工具窗口具有短于普通标题栏的标题栏和使用较小的字体绘制的窗口标题。 工具窗口不会显示在任务栏中，也不会显示在用户按 Alt+Tab 时出现的窗口中。|
|WS_EX_TOPMOST|指定使用此样式创建的窗口应放置在所有非最顶层窗口之上，并在关闭窗口时保持在窗口上方。 应用程序可使用 `SetWindowPos` 成员函数添加或删除此特性。|
|WS_EX_TRANSPARENT|指定使用此样式创建的窗口是透明的。 也就是说，该窗口下的任何窗口都不能被窗口遮盖。 使用此样式创建的窗口仅在更新其下的所有同级窗口后才会接收WM_PAINT消息。|
|WS_EX_WINDOWEDGE|指定窗口具有具有凸起边的边框。|
