# <a name="cmdichildwndex-class"></a>CMDIChildWndEx 类

该`CMDIChildWndEx`类提供 Windows 多个文档接口 （MDI） 子窗口的功能。 它扩展了[CMDIChildwnd类](../../mfc/reference/cmdichildwnd-class.md)的功能。 当 MDI 应用程序使用特定 MFC 类时，框架需要此类。

有关详细信息，请参阅位于 Visual Studio 安装的**VC\\\\atlmfc src\\mfc**文件夹中的源代码。

## <a name="syntax"></a>语法

```
class CMDIChildWndEx : public CMDIChildWnd
```

如果为 MDI 子级请求位图，则此参数为 TRUE，该子级当前处于活动状态且主窗口未最小化。 在这种情况下，默认处理将拍摄主窗口的快照。

*点位置*<br/>
指定位图在主（顶层）窗口客户端坐标中的位置。 此点应由被调用人提供。

### <a name="return-value"></a>返回值

如果处理，则将句柄返回到有效的 32bpp 位图，否则为 NULL。

### <a name="remarks"></a>备注

在派生类中重写此方法，并返回有效的 32bpp 位图，用于 MDI 子级的实时预览。 仅当 Windows 7 任务栏选项卡上显示 MDI 子项时，才会调用此方法。 如果返回 NULL，MFC 将调用默认处理程序，并使用`PrintClient`或`PrintWindow`获取位图。
