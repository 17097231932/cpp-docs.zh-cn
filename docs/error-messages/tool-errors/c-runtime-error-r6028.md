# <a name="c-runtime-error-r6028"></a>C 运行时错误 R6028

无法初始化堆

> [!NOTE]
> 如果在运行应用程序时遇到此错误消息，则该应用程序已关闭，因为它具有内部内存问题。 导致此错误的可能原因有很多，但通常是由于内存不足、程序中的 bug 或硬件驱动程序有缺陷导致的。
>
> 可以尝试以下步骤来修复此错误：
>
> - 关闭其他正在运行的应用程序或重新启动计算机以释放内存。
> - 使用 **控制面板** 中的 "**应用和功能**" 或 "**程序和功能**" 页来修复或重新安装该程序。
> - 如果应用在最近一次安装其他应用或驱动程序之前工作，请使用 **控制面板** 中的 "**应用和功能**" 或 "**程序和功能**" 页删除新应用或驱动程序，然后重试应用。
> - 请在 "**控制面板**" 中查看硬件供应商的网站或 **Windows 更新** 软件和驱动程序更新。
> - 检查应用的更新版本。 如果问题仍然存在，请与应用程序供应商联系。

**面向程序员的信息**

当操作系统未能为应用程序创建内存池时，将发生此错误。 具体来说，C 运行时 (CRT) 调用的 Win32 函数 `HeapCreate` 返回了指示失败的 NULL。

如果在应用程序启动期间发生此错误，则系统可能会因为加载了有缺陷的驱动程序而无法满足堆请求。 请在 Windows Update 或硬件供应商的网站中查看已更新的驱动程序。
