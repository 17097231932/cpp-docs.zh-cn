# 基于框架生成
[!INCLUDE[vs2017banner](../assembler/inline/includes/vs2017banner.md)]

在配置了应用程序的影响与 MFC 框架为提供特定的源代码和连接组件通过定义其响应的消息和命令。  使用 C\+\+ 语言标准 C\+\+ 和技术从类库提供的这些派生您自己的应用程序特定的类并重写和扩充基类的行为。  
  
 在相关主题，下表描述您通常遵循和职责和框架的职责的常规操作序列：  
  
-   [用于生成与 Framework 的应用顺序](../mfc/sequence-of-operations-for-building-mfc-applications.md)  
  
-   [用于创建 OLE 应用程序的操作顺序](../mfc/sequence-of-operations-for-creating-ole-applications.md)  
  
-   [用于创建 ActiveX 控件的操作顺序](../mfc/sequence-of-operations-for-creating-activex-controls.md)  
  
-   [用于创建数据库应用程序的操作顺序](../mfc/sequence-of-operations-for-creating-database-applications.md)  
  
 在很大程度上，可以遵循以下表作为一系列步骤创建的 MFC 应用程序，不过，某些步骤是可选定的方案。  例如，视图类的大多数应用程序使用类型从提供若干类型。
