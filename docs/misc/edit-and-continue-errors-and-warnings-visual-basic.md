# “编辑并继续”错误和警告 (Visual Basic)
利用 [!INCLUDE[vb_current_short](../misc/includes/vb_current_short_md.md)]“编辑并继续”，您可以在中断模式下停止程序执行，对执行代码进行更改，然后继续执行新合并了更改后的程序。  
  
 通常情况下，影响类的公共结构的声明性代码编辑是禁止的，但对类内部的方法、属性体或私有声明进行某些编辑是允许的。 “编辑并继续”会尽可能将不可编辑的代码标记为浅灰色并显示错误消息。 有关在编辑并继续中不支持的编辑详细信息 [!INCLUDE[vb_current_short](../misc/includes/vb_current_short_md.md)], ，请参阅 [Visual Basic“编辑并继续”中不支持的编辑](../Topic/Unsupported%20Edits%20in%20Visual%20Basic%20Edit%20and%20Continue.md)。  
  
 **更正“编辑并继续”错误**  
  
1.  在**“调试”**菜单上，选择**“撤消”**可撤消更改。  
  
     \- 或 \-  
  
2.  停止调试会话，进行编辑并启动新的调试会话。
