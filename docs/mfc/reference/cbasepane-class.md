以下自定义提示与 继承的`CBasePane Class`和 任何类有关：

- 创建窗格时，可以应用几种新样式：

  - AFX_CBRS_FLOAT使窗格浮动。

  - AFX_CBRS_AUTOHIDE启用自动隐藏模式。

  - AFX_CBRS_CLOSE使窗格能够关闭（隐藏）。

  这些是可以与位-OR 操作组合的标志。

`CBasePane`实现以下虚拟布尔方法以反映这些标志[：CBasePane：：可以关闭](#canbeclosed)[，CBasePane：：可以自动隐藏](#canautohide)[，CBasePane：：可以浮动](#canfloat)。 您可以在派生类中重写它们以自定义其行为。

- 您可以通过重写[CBasePane：：canAcceptPane](#canacceptpane)来自定义停靠行为。 让窗格从此方法返回 FALSE，以防止另一个窗格停靠到它。

- 如果要创建无法浮动的静态窗格，并且阻止任何其他窗格在它之前停靠（类似于 OutlookDemo 示例中的 Outlook 栏），请将其创建为非浮动，并重写[CBasePane：:DoesAllowDynInsert 之前](#doesallowdyninsertbefore)返回 FALSE。 如果创建窗格时没有AFX_CBRS_FLOAT样式，则默认实现将返回 FALSE。

- 使用 -1 以外的指示创建所有窗格。

- 要确定窗格可见性，请使用[CBasePane：：是可见的](#isvisible)。 它正确处理选项卡式和自动隐藏模式下的可见性状态。

- 如果要创建非浮动可调整大小的窗格，请创建它，而不AFX_CBRS_FLOAT样式，并调用[CFrameWnd：:DockControlBar](../../mfc/reference/cframewnd-class.md#dockcontrolbar)。

- 要从停靠布局中排除窗格或从停靠栏中删除工具栏，请调用[CBasePane：：：undockPane](#undockpane)。 不要为自动隐藏模式下的窗格或驻留在选项卡式窗口选项卡中的窗格调用此方法。

- 如果要浮动或取消停靠处于自动隐藏模式的窗格，则必须调用[CDockablePane：：在](../../mfc/reference/cdockablepane-class.md#setautohidemode)调用[CBasePane：：FloatPane](#floatpane)或[CBasePane：undockPane](#undockpane)之前，将 FALSE 设置为第一个参数。
