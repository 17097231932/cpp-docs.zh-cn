# “插入 ActiveX 控件”对话框
[!INCLUDE[vs2017banner](../assembler/inline/includes/vs2017banner.md)]

此对话框使您能够在使用[对话框编辑器](../mfc/dialog-editor.md)时[将 ActiveX 控件插入对话框](../mfc/viewing-and-adding-activex-controls-to-a-dialog-box.md)。  
  
 **ActiveX 控件**  
 显示 Active X 控件列表。  从此对话框插入控件并不生成包装类。  如果需要包装类，请使用[类视图](http://msdn.microsoft.com/zh-cn/8d7430a9-3e33-454c-a9e1-a85e3d2db925)创建（有关更多信息，请参见[添加类](../ide/adding-a-class-visual-cpp.md)）。  如果 Active X 控件没有出现在此对话框中，请按照供应商的说明尝试安装该控件。  
  
 **路径**  
 显示包含 ActiveX 控件的文件。  
  
 为了便于访问，可以将控件放在“工具箱”窗口中。  有关更多信息，请参见[“自定义工具箱”对话框](http://msdn.microsoft.com/zh-cn/bd07835f-18a8-433e-bccc-7141f65263bb)。  
  
 有关将资源添加到托管项目的信息，请参见“.NET Framework 开发员指南”中的[应用程序中的资源](../Topic/Resources%20in%20Desktop%20Apps.md)。有关手动将资源文件添加到托管项目、访问资源、显示静态资源和将资源字符串分配给属性的信息，请参见[演练：本地化 Windows 窗体](http://msdn.microsoft.com/zh-cn/9a96220d-a19b-4de0-9f48-01e5d82679e5)和[Walkthrough: Using Resources for Localization with ASP.NET](../Topic/Walkthrough:%20Using%20Resources%20for%20Localization%20with%20ASP.NET.md)。  
  
## 要求  
 Win32
