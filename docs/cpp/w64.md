# <a name="__w64"></a>__w64

此 Microsoft 特定的关键字已过时。 在早于 Visual Studio 2013 的 Visual Studio 版本中，这使你可以标记变量，以便在使用[/Wp64](../build/reference/wp64-detect-64-bit-portability-issues.md)进行编译时，编译器将报告在使用64位编译器进行编译时报告的任何警告。

## <a name="syntax"></a>语法

> *类型* **`__w64`***标识符*

### <a name="parameters"></a>参数

*type*<br/>
这三种类型中的一种可能导致代码从32位移植到64位编译器时出现问题： **`int`** 、 **`long`** 或指针。

*identifier*<br/>
要创建变量的标识符。

## <a name="remarks"></a>备注

> [!IMPORTANT]
> [/Wp64](../build/reference/wp64-detect-64-bit-portability-issues.md)编译器选项和 **`__w64`** 关键字在 Visual Studio 2010 中已弃用，并 Visual Studio 2013 并从 Visual Studio 2013 开始删除。 如果在 `/Wp64` 命令行中使用编译器选项，编译器将发出命令行警告 D9002。 **`__w64`** 关键字无提示地忽略。 不要使用此选项和关键字来检测64位可移植性问题，请使用面向64位平台的 Microsoft c + + 编译器。 有关详细信息，请参阅为[64 位 x64 目标配置 Visual C++](../build/configuring-programs-for-64-bit-visual-cpp.md)。

在 x86 上包含的任何 typedef **`__w64`** 和 x64 上的64位必须是32位。

若要使用早于 Visual Studio 2010 的 Microsoft c + + 编译器版本检测可移植性问题， **`__w64`** 应在更改大小为32位和64位平台之间的任何 typedef 上指定关键字。 对于任何此类类型， **`__w64`** 必须仅出现在 typedef 的32位定义上。

为了与早期版本兼容， **_w64** **`__w64`** 除非指定了编译器选项[/za " \( 禁用语言扩展](../build/reference/za-ze-disable-language-extensions.md)"，否则 _w64 是同义词。

**`__w64`** 如果编译不使用，则忽略关键字 `/Wp64` 。

有关移植到 64 位编译器的详细信息，请参阅以下主题：

## <a name="example"></a>示例

```cpp
// __w64.cpp
// compile with: /W3 /Wp64
typedef int Int_32;
#ifdef _WIN64
typedef __int64 Int_Native;
#else
typedef int __w64 Int_Native;
#endif

int main() {
   Int_32 i0 = 5;
   Int_Native i1 = 10;
   i0 = i1;   // C4244 64-bit int assigned to 32-bit int

   // char __w64 c;  error, cannot use __w64 on char
}
```
