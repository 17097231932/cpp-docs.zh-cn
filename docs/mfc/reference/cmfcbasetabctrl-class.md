# <a name="cmfcbasetabctrl-class"></a>CMFCBaseTabCtrl Class

实现选项卡式窗口的基本功能。
有关详细信息，请参阅位于 Visual Studio 安装的**VC\\\\atlmfc src\\mfc**文件夹中的源代码。

## <a name="syntax"></a>语法

```
class CMFCBaseTabCtrl : public CWnd
```

## <a name="customization-tips"></a>自定义提示

以下提示适用于 `CMFCBaseTabCtrl Class` 和从其继承的任何类：

- 如果启用可拆分选项卡，请勿保留指向选项卡式窗口的指针。 这些可拆分选项卡可以动态创建和销毁。 因此，指针可能会变为无效。

- 可以配置选项卡控件，使用户可以通过使用鼠标动态移动选项卡控件上的选项卡。 此功能已内置于 `CMFCBaseTabCtrl` 类。 调用 [CMFCBaseTabCtrl::EnableTabSwap](#enabletabswap)可启用它。

- 默认情况下，将选项卡添加到选项卡控件时，选项卡是可拆分的。 还可以使用 [CMFCBaseTabCtrl::AddTab](#addtab)来添加非可拆分的选项卡。 如果将 `bDetachable` 参数设置为 `FALSE`，则选项卡将不可拆分。 还可调用 [CMFCBaseTabCtrl::EnableTabDetach](#enabletabdetach)方法来更改选项卡是否可拆分。

- 从[CWnd 类](../../mfc/reference/cwnd-class.md)派生的对象可以放在可停靠的控制栏或可停靠选项卡上。要使整个控件停靠，必须使`CWnd`对象可停靠。 MFC 使用一个包装类实现此目的。 此包装类是 [CDockablePaneAdapter Class](../../mfc/reference/cdockablepaneadapter-class.md)。 添加到可停靠控件条或可停靠选项卡上的任何 `CWnd` 对象将被包装到 `CDockablePaneAdapter` 对象中。 可以通过将 `m_bEnableWrapping` 对象的 `CMFCBaseTablCtrl` 参数设置为 `FALSE`来禁用自动包装。 还可以使用 [CMFCBaseTabCtrl::SetDockingBarWrapperRTC](#setdockingbarwrapperrtc)方法来更改应用程序将用作包装器的类。

## <a name="cmfcbasetabctrlshowtab"></a><a name="showtab"></a>CMFCBaseTabctrl：：显示选项卡

显示或隐藏指定的选项卡。

```
virtual BOOL ShowTab(
    int iTab,
    BOOL bShow = TRUE,
    BOOL bRecalcLayout = TRUE,
    BOOL bActivate = FALSE);
```

### <a name="parameters"></a>参数

*iTab*<br/>
[在]`ShowTab`将显示或隐藏选项卡的索引。

*b显示*<br/>
[在]指示是否显示选项卡的布尔参数。

*bRecalcLayout*<br/>
[在]指示是否立即重新计算窗口布局的布尔参数。

*b 激活*<br/>
[在]指示是否选择*iTab*指定的选项卡的布尔参数。

### <a name="return-value"></a>返回值

如果成功，则不为 0；否则为 0。

### <a name="remarks"></a>备注

仅当*bShow*为 TRUE 时，参数*bActivate*才适用。 如果*bActivate*为`ShowTab`TRUE，并且`ShowTab`成功，则将消息AFX_WM_CHANGE_ACTIVE_TAB发送到选项卡窗口的父级。
