# <a name="compiler-warning-level-4-c4464"></a>编译器警告 （等级 C4464

> **相对包含路径包含..**

一个`#include`指令具有的路径包含.. 父目录说明符。

## <a name="remarks"></a>备注

从 Visual Studio 2015 Update 1 开始，因此编译器可检测 include 指令包含.. 路径段和问题警告，如果已启用。 以这种方式编写的代码通常用于包含因不正确使用项目相对路径而留在项目外的标头。 这将创建可以通过不是编程人员计划，包括不同的源代码文件编译该程序或这些相对路径将不能移植到其他生成环境的风险。 虽然没有特定的警告，我们还建议您不要使用父目录路径段，以指定你的项目包含目录。

此警告是 Visual Studio 2015 Update 1 中的新增功能，默认情况下处于关闭状态。 使用[/wall](../../build/reference/compiler-option-warning-level.md)若要启用默认情况下处于关闭状态的所有警告或 __/w__*n*__4464__启用作为级别 C4464 *n*警告。 有关详细信息，请参阅[编译器警告，是 Off By Default](../../preprocessor/compiler-warnings-that-are-off-by-default.md)。 有关如何禁用警告的编译器版本的信息，请参阅[由编译器版本的编译器警告](compiler-warnings-by-compiler-version.md)。

## <a name="example"></a>示例

源代码文件，使用.. 路径段可以触发此警告 **/wall**指定选项：

```cpp
#include "..\headers\C4426.h"  // emits warning C4464

// To fix this issue, specify only the header file name, and add
// the absolute path to 'headers\' to your project's include directories
#include "C4426.h"
```
