# <a name="compiler-warning-level-4-c4596"></a>编译器警告 (等级 4) C4596

> "*name*"：成员声明中的非法限定名

## <a name="remarks"></a>备注

成员声明具有意外限定。 若要解决此警告，请从标识符中删除限定。

默认情况下，此警告处于关闭状态。 可以使用 [/Wall](../../build/reference/compiler-option-warning-level.md) 或 __/w__*N*__4596__ 在命令行上将其启用为级别 *N* 警告。 或者，在源文件中使用 (*N*： 4596) [#pragma 警告](../../preprocessor/warning.md)。 有关详细信息，请参阅 [默认情况下处于关闭状态的编译器警告](../../preprocessor/compiler-warnings-that-are-off-by-default.md)。 某些版本的编译器仅在 [/permissive-](../../build/reference/permissive-standards-conformance.md)下生成此警告。

从 Visual Studio 2015 Update 3 开始提供此警告。 在早期版本的编译器中，在早期版本的编译器中编译的代码现在可以生成 **C4596**。 有关如何禁用特定编译器版本或更高版本中引入的警告的信息，请参阅编译器 [警告（按编译器版本](compiler-warnings-by-compiler-version.md)）。

## <a name="example"></a>示例

此示例生成 C4596，并演示如何修复此问题：

```cpp
// C4596.cpp
// compile with: /w14596 /c

struct A {
    void A::f() { } // error C4596: illegal qualified name in member
                    // declaration.
                    // Remove redundant 'A::' to fix.
};
```
