# MFC DLL 向导
[!INCLUDE[vs2017banner](../../assembler/inline/includes/vs2017banner.md)]

当使用 MFC DLL 向导创建 MFC DLL 项目时，得到一个具有内置功能的工作起始应用程序，该功能编译后将实现 [DLL](../../build/dlls-in-visual-cpp.md) 的基本功能。  MFC 起始程序包括 C\+\+ 源文件 \(.cpp\)、资源文件 \(.rc\) 和一个项目文件 \(.vcxproj\)。  这些起始文件中生成的代码基于 MFC。  有关更详细的信息，请参见 Visual Studio 和 [MFC DLL 向导生成的类和函数](../../mfc/reference/classes-and-functions-generated-by-the-mfc-dll-wizard.md)中为项目生成的 Readme.txt 中的文件详细信息。  
  
## 概述  
 该向导页描述正在创建的当前 [MFC DLL 项目的应用程序设置](../../mfc/reference/application-settings-mfc-dll-wizard.md)。  默认情况下，项目被创建为没有附加设置的规则 DLL（共享 MFC）项目。  
  
 若要更改这些默认设置，单击向导左列中的**“应用程序设置”**并在此 MFC DLL 向导页中进行更改。  
  
 创建 MFC DLL 项目后，可以使用 Visual C\+\+  [代码向导](../../ide/adding-functionality-with-code-wizards-cpp.md)向项目中添加对象或者控件。  
  
 可以执行基本 MFC DLL 项目的下面增强任务和类型：  
  
-   [从 DLL 导出](../../build/exporting-from-a-dll.md)  
  
-   [将可执行文件链接到 DLL](../../build/linking-an-executable-to-a-dll.md)  
  
-   [初始化 DLL](../../build/initializing-a-dll.md)
